これらの記事を参考に、つまづきながらLチカできた。この知見をまとめ、以下にWinsows11を用いてRaspberry Pi Picoにmicropythonを書き込みLチカする方法をなるべく簡潔に具体的に書いた。


以下Raspberry Pi PicoをPicoと略す

## picoにmicropythonのファームウェアを書き込む

- PicoのBOOTSELボタンを押しながらPCとpicoをデータ転送に対応したUSB-Micro USB Type-Bケーブルで接続する。picoがブートモードでPCに接続され、USBメモリを接続したときのように、エクスプローラー上で「PC」の欄に「RPI-RP2」みたいな名前でpicoが認識される。
- [https://www.raspberrypi.com/documentation/microcontrollers/micropython.html]
のDownload the correct MicroPython UF2 file for your board: Picoの「Pico」をクリックして「RPI_PICO-20250415-v1.25.0.uf2」のようなmicro python用のuf2ファイルをダウンロードする。
- ダウンロードしたuf2ファイルをエクスプローラー上でRPI-RP2にドラッグする。uf2がpicoにコピーされ、コピーが完了するとファームウェアの読込が始まり、picoがエクスプローラー上から消える。
- picoをusbケーブルから外し再度接続する。picoが通常モードでPCに接続され、thonnyによるデバッグが可能になる。


## picoにmicropythonのコードを書きこむ
- [https://thonny.org/]の「Download version 4.1.7 for Windows」といった記述の「windows」をクリックして「thonny-4.1.7.exe」といった名前のexeファイルをダウンロードする。ダウンロードしたexeファイルをダブルクリックしてthonnyをインストールする。
- thonnyを起動し、run>configure interpreterから出る選択メニューで「which kind of interpreter should Thonny use for running your code?」を「MicroPython (Raspberry Pi Pico)」にし、「Port or WevREPL」を「< Try to detect port automatically >」にし、okをクリックして閉じる。thonnyがpicoに接続しデバッグ状態になる。
- 以下のpythonコードをthonnyの編集画面に張り付ける。

```
from machine import Pin
import utime
led = Pin(25, Pin.OUT)
while True:
    led.value(1)
    utime.sleep(0.1)
    led.value(0)
    utime.sleep(0.1)
```
- (「import machine」にすると「NameError: name 'Pin' isn't defined」エラーが出てしまう)
[https://stackoverflow.com/questions/61365990/receiving-nameerror-name-pin-isnt-defined-when-running-python-code-on-an-e]

- 緑の再生マークをクリックしてコードを実行するとLEDがちかちか点滅する(Lチカ)。このようにthonny上でコードのデバッグができる。赤のSTOPボタンをクリックしてコードの実行を止める。
- File>save a copyから開く保存ビューでRaspberry Pi Picoを選択し"main.py"という名前で保存する。コードがpicoに書き込まれる。
- Picoを取り外して、PCとは別のUSB電源とつなぐと、デバッグと同様にLチカするようになる。
